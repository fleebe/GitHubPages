Dependency Graph TS

Dependency Graph TS is a command-line tool designed to analyze TypeScript and JavaScript projects. It generates dependency graphs, visualizes module exports and class structures, and integrates with TypeDoc and JSDoc to produce comprehensive documentation and an HTML report.

Features
  • Dependency Analysis: Scans your project to understand module relationships.
  • Graph Generation: Creates DOT files representing:
    • Module-to-module dependencies.
    • Directory-level dependencies.
    • Node module dependencies.
    • (Optional) Detailed export lists for each module.
    • (Optional) Class diagrams showing properties, methods, and inheritance.
  • SVG Visualization: Converts DOT files into SVG images for easy viewing.
  • Documentation Integration:
    • Generates TypeScript documentation using TypeDoc.
    • Generates JavaScript documentation using JSDoc.
  • HTML Report: Produces a central index.html file that links to all generated diagrams and documentation, providing a unified view of your project structure.
  • Configurable: Supports custom configuration files for TypeDoc and JSDoc, and allows fine-grained control over included/excluded files.

Prerequisites

Before you begin, ensure you have the following installed:

  • Node.js (which includes npm)
  • Graphviz (for converting DOT files to SVG). Make sure dot is in your system's PATH.
  • TypeDoc (globally or as a project dependency if generating TypeScript docs): npm install -g typedoc
  • JSDoc (globally or as a project dependency if generating JavaScript docs): npm install -g jsdoc

Installation

  1. Clone the repository:

    git clone <your-repository-url>
    cd dep-graph-ts

  2. Install dependencies:

    npm install

  3. Build the project:

    npm run build

    This will compile the TypeScript code into the dist directory.

  4. (Optional) Link for global CLI usage: If you want to use dep-graph-ts as a global command:

    npm link

    You can then run dep-graph-ts from any directory. Otherwise, you can run it using node dist/dep-graph.js.

Usage

The basic command structure is:

dep-graph-ts <source_directory_or_file> [options]

Or, if not linked globally:

node dist/dep-graph.js <source_directory_or_file> [options]

Arguments:

  • <source_directory_or_file>: The path to the TypeScript/JavaScript project directory or a single entry file to analyze.

Options:

  • -o, --output <directory>: Specify the output directory for generated files (default: ./dep_graph_output).
  • -c, --class: Generate class diagrams.
  • -e, --export: Generate module export graphs.
  • -d, --docs: Generate source code documentation using TypeDoc (for TS) and JSDoc (for JS).
  • --typedoc-config <path>: Path to a custom TypeDoc configuration file (e.g., typedoc.json).
  • --jsdoc-config <path>: Path to a custom JSDoc configuration file (e.g., jsdoc.json).
  • --exclude <patterns...>: Glob patterns for files/directories to exclude from analysis (e.g., "**/node_modules/**", "**/*.test.ts").
  • --include <patterns...>: Glob patterns for files/directories to explicitly include. If used, only matching files are processed.
  • -h, --help: Display help for command.

Example:

dep-graph-ts ./my-project -o ./docs_output -c -e -d --typedoc-config ./typedoc.config.json

Configuration

TypeDoc & JSDoc

You can customize the behavior of TypeDoc and JSDoc by providing your own configuration files using the --typedoc-config and --jsdoc-config options respectively.

  • TypeDoc: Typically a typedoc.json file. See TypeDoc documentation for available options. dep-graph-ts will respect the out directory specified in this file if present.
  • JSDoc: Typically a jsdoc.json or conf.json file. See JSDoc documentation for configuration details. dep-graph-ts will respect the opts.destination specified in this file if present.

If no configuration files are provided, dep-graph-ts will use sensible defaults and generate documentation into subdirectories (typedoc_out and jsdoc_out) within the main output directory.

Output

All generated files will be placed in the specified output directory (default: dep_graph_output). The structure will typically look like this:

output_directory/
├── index.html                # Main HTML report linking everything
├── diagrams.html             # Page displaying all generated SVG diagrams
├── ModuleDependencies.dot    # DOT source for module dependencies
├── ModuleDependencies.svg    # SVG image of module dependencies
├── DirectoryDependencies.dot # DOT source for directory dependencies
├── DirectoryDependencies.svg # SVG image of directory dependencies
├── NodeModules.dot           # DOT source for external node module dependencies
├── NodeModules.svg           # SVG image for external node module dependencies
├── (if -e) ExportGraph_<dirname>.dot / .svg  # Export graphs per directory
├── (if -c) ClassGraph_<dirname>.dot / .svg   # Class graphs per directory
├── (if -d) typedoc_out/      # TypeDoc generated documentation
│   └── index.html
│   └── ...
└── (if -d) jsdoc_out/        # JSDoc generated documentation
    └── index.html
    └── ...
  • index.html: The main entry point to view your project's structure, diagrams, and documentation.
  • diagrams.html: A dedicated page that embeds all generated SVG diagrams for easier browsing.
  • .dot files: Graphviz source files. You can manually process these with dot if needed.
  • .svg files: Scalable Vector Graphics images of your project's dependencies and structures.
  • typedoc_out/: Contains the HTML documentation generated by TypeDoc for your TypeScript files.
  • jsdoc_out/: Contains the HTML documentation generated by JSDoc for your JavaScript files.
